# 購入依頼リスト整形スキル

松本十帖の週次発注依頼を作成するための整形手順。商品名・数量（×個数）・設置場所・購入URLの羅列を、下記フォーマットへ変換する。

## 入力データの形式

1商品につき、おおむね以下の情報が与えられる（順序・改行は不定）。

- 商品名（メーカー名・型番・容量などを含む長い名称）
- `×個数`（発注数量）と設置場所（例：ダイニング / 清掃 / 哲学 / ショップ）
- 購入サイトのURL

## 整形ルール

1. **購入サイトごとに並べ替える。** URLのドメインでサイトを判定し、同じサイトの商品をまとめる。並び順は原則 `羽車 → アスクル → 楽天市場 → Amazon`。

2. **サイト見出しは日本語通称に★を付ける。** ドメイン表記（`askul.co.jp` など）は括弧書きも含めて記載しない。
   - `haguruma.co.jp` → `★羽車`
   - `askul.co.jp` → `★アスクル`
   - `item.rakuten.co.jp` → `★楽天市場`
   - `amzn.asia` / `amazon.co.jp` → `★Amazon`
   - 見出しは Markdown の `## ★サイト名` とする。

3. **各サイト内は番号付きリスト**（`1.` `2.` …）とし、**商品名は太字**（`**商品名**`）にする。

4. **商品名の後の「×個数 設置場所」は必ず改行する。** すべての商品で以下の3行構成に統一する。

   ```
   1. **商品名**
      ×個数 設置場所
      URL
   ```

5. **冒頭に依頼文テンプレートを付ける。**

   ```
   【購入依頼 松本十帖 YYYY年M月D日】
   @長谷川瞳
   おつかれさまです。
   今週の発注依頼です。
   よろしくお願いいたします。
   ```

   - 日付は西暦。スラッシュは使わず `YYYY年M月D日` 形式（例：`2026年7月15日`）。作成日（当日）を入れる。

6. **末尾に集計（合計点数など）は付けない。**

## 出力例

```
【購入依頼 松本十帖 2026年7月15日】
@長谷川瞳
おつかれさまです。
今週の発注依頼です。
よろしくお願いいたします。

## ★アスクル

   ×3 清掃

```

## 注意点

- 商品名内に含まれる `×`（例：`10枚入り×24個`）や `【】`（例：`【公式】`）は商品名の一部なので削除しない。除去するのは購入サイトを示す括弧書きのドメイン表記のみ。
- 数量の `×個数` は必ず商品名とは別の行に置く。
